ALERT ROUTING & NOTIFICATIONS
================================================================================

Alert Categories with Routing:

CRITICAL ALERTS → Immediate dispatch (0s wait)
├── Destination: #critical-alerts (Slack)
├── Escalation: PagerDuty (on-call page)
├── Examples: Service down, high error rate (>10%), critical latency
└── Response: Page on-call, P1 incident

SERVICE TEAM ALERTS → Service team routing
├── Destination: #service-team (Slack)
├── Grouping: By service name
├── Wait: 10 seconds
└── Examples: HTTP errors, latency issues

DATABASE TEAM ALERTS → Database routing
├── Destination: #database-team (Slack)
├── Grouping: By instance
├── Includes: Grafana dashboard link
└── Examples: Connection pool, slow queries

INFRASTRUCTURE TEAM ALERTS → Infrastructure routing
├── Destination: #infrastructure-team (Slack)
├── Grouping: By instance
├── Wait: 30 seconds
└── Examples: CPU, memory, disk usage

CACHE TEAM ALERTS → Cache routing
├── Destination: #cache-team (Slack)
├── Wait: 10 seconds
└── Examples: Redis down, high memory, evictions

MESSAGING TEAM ALERTS → Messaging routing
├── Destination: #messaging-team (Slack)
├── Examples: Kafka consumer lag
└── Wait: 10 seconds

SECURITY TEAM ALERTS → Security escalation
├── Destination: #security-team (Slack)
├── Escalation: PagerDuty (urgent)
├── Examples: SSL cert expiry
└── Response: Immediate action required

INFO ALERTS → Informational tracking
├── Destination: #monitoring-info (Slack)
├── Wait: 30 minutes (batched)
└── Examples: Cache hit rate low, idle connections

NOTIFICATION CHANNELS SUPPORTED:
✓ Slack (webhook-based)
✓ PagerDuty (incident creation)
✓ OpsGenie (alert management)
✓ Email (SMTP-based)

ALERT INHIBITION (Smart Suppression):
✓ Critical suppresses warning/info for same service
✓ Database down suppresses connection pool alerts
✓ Redis down suppresses memory/eviction alerts
✓ Platform-wide issues suppress service-specific alerts

PER SERVICE INTEGRATION:

Minimal Changes Required (1 import + 2 function calls):

1. Add import to main.py:
   from shared.middleware.metrics import PrometheusMiddleware, setup_metrics_routes

   app.add_middleware(PrometheusMiddleware, app_name="service-name")

3. Setup metrics endpoint:
   setup_metrics_routes(app)
