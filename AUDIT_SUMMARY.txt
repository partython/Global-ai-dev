================================================================================
SECURITY IMPROVEMENTS
================================================================================

BEFORE → AFTER

1. Key Injection
   BEFORE: tenant_id = "admin:*" → escapes key structure
   AFTER:  tenant_id = "admin:*" → ValueError: Invalid format

2. Rate Limiter
   BEFORE: Redis timeout → allow all requests (fail-open)
   AFTER:  Redis timeout → block all requests (fail-closed)

3. Kafka Security
   BEFORE: Default PLAINTEXT (no encryption, no auth)
   AFTER:  Default SASL_SSL + SCRAM-SHA-512 (TLS + strong auth)

4. Kafka Credentials
   BEFORE: SASL_SSL possible without credentials (silent failure at runtime)
   AFTER:  Validates at module import (fast-fail with clear error)

5. Replication
   BEFORE: Hard-coded replication_factor = 1 (single failure = data loss)
   AFTER:  Configurable via KAFKA_MIN_REPLICATION (default: 3, production-grade)

6. Error Handling
   BEFORE: Silent exception swallowing in lock release and deserialization
   AFTER:  Detailed logging with context (offset, partition, tenant_id)

ENVIRONMENT VARIABLES TO SET:

For Kafka (Production):

  export KAFKA_MIN_REPLICATION=1

For Redis:

  export REDIS_PASSWORD=<password-if-required>
