FTC-001: Insecure Credential Encryption (XOR Cipher)
  File:     /services/tenant_config/main.py (lines 1335-1364)
  Severity: CRITICAL
  Status:   ✓ FIXED

  Problem:  Used simple XOR instead of authenticated encryption
  Impact:   Attackers could tamper with encrypted credentials undetected

  Fix:      Replaced with AES-256-GCM (AEAD cipher with authentication)
  - Derived 256-bit key using PBKDF2 (100,000 iterations)
  - Uses 12-byte random nonce per encryption
  - Authentication tag prevents tampering
  - Requires: pip install cryptography>=41.0.0

  Lines Changed: ~50 lines in _encrypt_credential() and _decrypt_credential()

FTC-006 to FTC-011: Error Detail Leakage (10 instances in email/main.py)
  File:     /services/email/main.py (lines 471, 662, 686, 714, 738, 759, 785, 847, 875, 912, 1032)
  Severity: HIGH
  Status:   ✓ FIXED (all 11 instances)

  Problem:  HTTP error responses included str(exception) - full details exposed

  Vulnerable Endpoints:
  - POST /webhook/ses
  - POST /api/v1/send
  - GET/POST/PUT/DELETE /api/v1/templates (4 instances)
  - GET/POST /api/v1/domains (2 instances)
  - POST /webhook/ses/notifications
  - GET /api/v1/analytics

  Attack Example:
    Response: {"error": "IntegrityError: UNIQUE constraint failed on verified_domains"}
    ↓ Attacker learns about database schema, table names, constraints

  Fix:      All exceptions now return generic error messages
  - Details logged internally: logger.error(f"Error sending email: {e}")
  - Client receives: {"error": "Failed to send email"}

  Lines Changed: 11 error handlers + comments
