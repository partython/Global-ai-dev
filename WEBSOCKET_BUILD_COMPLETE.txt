KEY FEATURES
================================================================================

✅ Multi-Instance Support
   • Redis pub/sub for cross-instance messaging
   • Stateless service design
   • Automatic message distribution

✅ Tenant Isolation
   • Connections scoped to single tenant
   • Cannot cross tenant boundaries
   • Validated on every operation

✅ Connection Management
   • Per-user connection tracking
   • Connection limits by plan tier
   • Graceful connect/disconnect lifecycle

✅ Room-Based Broadcasting
   • Conversation rooms: conversation:{id}
   • Dashboard rooms: dashboard:{tenant_id}
   • Agent rooms: agent:{id}
   • Tenant rooms: tenant:{tenant_id}
   • Metric rooms: metric:{type}

✅ Heartbeat Protocol
   • 30-second ping/pong interval
   • Keeps connections alive
   • Detects dead connections

✅ Security
   • JWT authentication required
   • Token expiry validation
   • Role-based access control
   • Tenant isolation enforcement

✅ Metrics & Monitoring
   • Real-time connection metrics
   • Message counts and rates
   • Error tracking
   • Per-tenant and per-room statistics

✅ Message Types
   • Chat messages (with persistence)
   • Typing indicators
   • Presence updates
   • Conversation events (assigned, closed)
   • Agent status updates
   • Dashboard metrics
   • Notifications and alerts
   • System messages

ARCHITECTURE
================================================================================

CLIENT (Browser/Mobile)
    ↓ WebSocket
    ↓ JWT Auth
GATEWAY (Port 9001)
    ├─ ws/conversations/{id}
    ├─ ws/dashboard
    └─ ws/agent
    ↓ Redis Pub/Sub
REDIS (Pub/Sub Backend)
    ↓
CONVERSATION SERVICE (Port 9004)
    ├─ REST API
    └─ ws/live/{id} (with persistence)
    ↓
SERVICES (Channel Router, Notification, etc.)
    ↓ Event Bus
    ↓ Redis Pub/Sub

CONNECTION LIMITS BY PLAN
================================================================================

Free:          5 connections per user
Starter:      25 connections per user
Growth:      100 connections per user (default)
Professional: 500 connections per user
Enterprise: 1000 connections per user

Enforced at: tenant_id + user_id level
Returns: 403 error if limit exceeded

================================================================================
WEBSOCKET ENDPOINTS
================================================================================

GATEWAY (Port 9001)
  ws://api.priyaai.com/ws/conversations/{conversation_id}
    • Authentication: Authorization: Bearer {JWT}
    • Room: conversation:{id}
    • Messages: chat, typing, presence

  ws://api.priyaai.com/ws/dashboard
    • Authentication: Authorization: Bearer {JWT}
    • Room: dashboard:{tenant_id}, metric:{type}
    • Messages: metrics_update, subscribe, unsubscribe

  ws://api.priyaai.com/ws/agent
    • Authentication: Authorization: Bearer {JWT}
    • Room: agent:{id}, agent_queue:{tenant_id}
    • Messages: agent_status, presence

  GET /ws/metrics
    • Returns: Connection and message metrics

CONVERSATION SERVICE (Port 9004)
  ws://conversation:9004/ws/live/{conversation_id}
    • Authentication: ?token={JWT}
    • Room: conversation:{id}
    • Messages: chat (persisted), typing
    • Features: Full message persistence, event publishing
